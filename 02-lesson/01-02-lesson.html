New York Air Quality Data

The New York air quality data contain daily air quality measurements in New York from May to September 1973. The data have 153 observations and 6 variables.

Each row is one day. The variables include ozone level, solar radiation, wind speed, temperature, month, and day.

You can read more about the original data description here: https://stat.ethz.ch/R-manual/R-devel/library/datasets/html/airquality.html

The Pipe Operator

The pipe operator sends the result from one step into the next step. You can read it as “then.”

df %>% filter(Month == 5) means “start with df, then select all rows where Month equals 5.”

df %>% 
  filter(Month == 5)

The native R pipe |> works similarly.

df |> 
  select(Ozone, Wind, Month) |>
  filter(Month == 5)

dplyr Data Manipulation Functions

Common dplyr functions include:

  • filter() chooses rows based on column values.
  • slice() chooses rows based on row location.
  • arrange() changes the order of rows.
  • select() chooses columns.
  • rename() changes column names.
  • mutate() changes existing columns or creates new columns.
  • relocate() changes the order of columns.
  • summarise() collapses rows into summaries.

Creating a New Column

Use mutate() to create a new column or modify an existing column.

For example, we can create a temperature category.

df2 <- df %>%
  mutate(Temp_Category = ifelse(Temp > 80, "High", "Moderate"))

head(df2)

Contingency Table

A contingency table shows counts for combinations of categorical variables.

First, we create a categorized ozone variable. Then we count the number of days in each combination of Month and Ozone_Level.

df <- df %>%
  mutate(Ozone_Level = ifelse(Ozone > median(Ozone, na.rm = TRUE),
                              "High", "Low"))

contingency_table <- df %>%
  group_by(Month, Ozone_Level) %>%
  summarise(Count = n()) %>%
  pivot_wider(names_from = Ozone_Level, values_from = Count)

print(contingency_table)

Contingency Table with a Categorized Variable

We can also categorize Solar.R:

Solar.R <= 115       : Low
115 < Solar.R <= 185 : Medium
258 < Solar.R        : High
contingency_table2 <- df %>%
  mutate(solar_cat = case_when(
    Solar.R <= 115 ~ "Low",
    Solar.R > 115 & Solar.R <= 185 ~ "Medium",
    Solar.R > 258 ~ "High"
  )) %>%
  group_by(solar_cat, Ozone_Level) %>%
  summarise(Count = n()) %>%
  pivot_wider(
    names_from = Ozone_Level,
    values_from = Count
  )

print(contingency_table2)

Filtering Out Missing Values

Some variables contain missing values, shown as NA.

head(df)

To remove rows where a specific column is missing, combine filter() with !is.na().

filtered_df <- df %>%
  filter(!is.na(Ozone), !is.na(Solar.R))

print(filtered_df[1:10, ])

To remove rows that have missing values in any column, use drop_na().

df_clean <- df %>%
  drop_na()

head(df_clean)
